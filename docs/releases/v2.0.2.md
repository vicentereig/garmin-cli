# v2.0.2 Release Notes

## Summary

`v2.0.2` is a patch release that finishes the mobile JSON auth fix by using API-specific headers for login and MFA requests, which restores successful login behavior after Garmin's latest SSO changes.

## User-Facing Changes

- `garmin auth login` now uses browser-style headers only for the initial sign-in page request.
- Login and MFA verification requests now use API/mobile-style headers that match Garmin's JSON auth endpoints.
- Failed login responses now include more response-body context to make auth failures easier to diagnose.

## Internal Cleanup

- Removed leftover alternate-domain runtime usage from the Garmin API and SSO clients.
- Stopped surfacing the stored auth domain in `garmin auth status`.
- Removed dead custom-domain unit tests tied to the old `garmin.cn` path.

## Suggested Release Checklist

1. Review this release note and edit the wording you want on the GitHub release page.
2. Tag the release:

```bash
git tag -a v2.0.2 -m "v2.0.2"
git push origin v2.0.2
```

3. Confirm the `release` workflow publishes artifacts, pushes to crates.io, and updates `vicentereig/homebrew-tap`.
4. Verify install paths after the workflow finishes:

```bash
brew install vicentereig/tap/garmin
garmin auth login
garmin auth status
```
