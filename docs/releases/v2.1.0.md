# v2.1.0 Release Notes

## Summary

`v2.1.0` is a minor release that adds note workflows for activities and sleep, improves synced activity data quality, and captures more track-point detail from GPX exports.

## User-Facing Changes

- Added `garmin activities note get`, `set`, and `clear` for reading and editing activity notes. Garmin stores these as the activity `description`.
- `garmin health sleep` now shows the Garmin mobile app sleep note when the sleep response includes `dailySleepDTO.userNote`.
- FIT downloads are now named and documented as Garmin FIT archives.

## Sync and Data Lake Improvements

- Sleep notes are now persisted as a typed `sleep_note` column in synced daily-health Parquet files.
- GPX sensor extension streams are now parsed into track points, preserving more per-point sensor data during sync.
- Calories, average heart rate, and max heart rate now parse correctly when Garmin returns those JSON fields as floats.

## Contributors

- Thanks to peetzweg for the sleep-note and activity-note features in PRs #11, #12, and #13.
- Thanks to Mark Woodbridge for the activity metric parsing fix in PR #14.

## Notes

- Existing daily-health Parquet files without `sleep_note` remain readable. Historical notes appear after a re-sync or backfill.
- Activity note writes are covered by tests, but the PUT payload shape still deserves live-account verification before depending on it for critical workflows.


